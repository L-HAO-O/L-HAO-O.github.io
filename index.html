如何用 hexo 和 github 搭建个人博客

如果你想拥有一个自己的博客网站，但是又不想花钱购买服务器和域名，那么你可以考虑使用 hexo 和 github 来搭建一个免费的静态博客网站。hexo 是一个基于 Node.js 的快速、简洁且高效的博客框架，它可以将你用 Markdown 语法写的文章渲染成漂亮的 HTML 网页。github 是一个提供免费代码托管服务的平台，它也提供了一种叫做 github pages 的功能，可以让你在 github 上发布和托管你的静态网站。通过将 hexo 生成的网页部署到 github pages 上，你就可以拥有一个完全免费且稳定的个人博客网站了。

本文将详细介绍如何使用 hexo 和 github 搭建个人博客网站的步骤，包括准备工作、安装和配置 hexo、部署到 github pages、绑定域名等。本文假设你已经有了一个 github 账号，并且对 git 和 Markdown 有一定的了解。如果不熟悉，可以先学习一下相关的教程。

步骤一：安装HEXO
首先需要通过NPM安装HEXO，具体操作如下：

打开终端（MacOS或Linux）或者命令行（Windows）；
输入以下命令安装HEXO：
npm install -g hexo-cli
步骤二：初始化博客
在安装完HEXO之后，我们需要创建一个博客文件夹，并初始化博客。具体操作如下：

在终端或命令行中输入以下命令，创建博客文件夹：
mkdir 你的博客名字
进入博客文件夹，初始化博客：
cd 你的博客名字
hexo init
步骤三：设置配置文件
了解下博客的配置文件夹_config.yml，包含如下内容：

Site
title：博客网站名；
subtitle：博客网站副标题；
description：网站描述；
author：博客作者；
language：语言；
timezone：时区；
URL
permalink：文章地址的格式；
root：网站根路径；
permalink_defaults：文章地址默认格式；
Directory
source：源文件目录，包含Markdown文件等；
public：公共文件生成目录；
tag_dir：标签页面目录；
archive_dir：时间线目录；
category_dir：分类页面目录；
Writing
new_post_name：新建文章的文件名格式；
default_layout：默认页面布局；
titlecase：标题首字母是否大写；
external_link：是否自动跳转；
filename_case：文件名格式；
render_drafts：未发布的文档的渲染策略；
Deployment：部署相关设置
在博客文件夹中打开 _config.yml 文件，修改里面的几个主要配置：

Site

title: 你的博客标题
description: 你的博客的描述

URL

url: https://你的github名字.github.io
permalink: :year/:month/:day/:title/

Deployment

deploy:
type: git
repo: git@github.com:你的github名字/你的github名字.github.io.git
branch: master
步骤四：编写文档
在博客文件夹中输入以下命令，用Hexo创建文章：

hexo new “文章标题”
新文章在博客文件夹中的 /source/_posts/ 文件夹中生成，打开 .md 文件，在文件中编写文章。

步骤五：生成静态文件
在博客文件夹中，输入以下命令，生成静态文件：

hexo generate
然后Hexo将会通过 /themes/ 中的主题文件生成静态文件，并拷贝到 /public/ 中。

步骤六：将博客提交到GitHub
在博客文件夹中，输入以下命令，将博客提交到GitHub：

hexo deploy
具体操作可以参考上面的 _config.yml 文件。如果你能够成功部署你的博客，那么你可以在 https://你的github名字.github.io 链接上查看到你的博客了。

步骤七：维护与更新
Hexo提供很多命令来维护和更新你的博客，比如：

hexo clean：清除生成的 public 文件夹和缓存文件；
hexo new “文章标题”：新建一篇博客文章；
hexo generate：生成静态页面；
hexo server：本地预览博客；
hexo deploy：将博客配置部署到GitHub。
结论
使用HEXO和GITHUB搭建个人博客不仅简单快捷，而且可以让你更好地展示你的想法和创意。希望这篇文章能够帮助你快速入门，让你能够拥有自己的个人博客。
